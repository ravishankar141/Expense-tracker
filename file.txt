❯  what is this project all about                                               

● Based on my exploration, Spendly is a personal expense tracking web
  application designed as an educational project.

  Purpose

  It's a Flask-based web app where users can:
  - Register and log in
  - Track their income and expenses
  - Categorize transactions (Food, Transport, Entertainment, etc.)
  - View financial summaries

  Educational Design

  This is a learning project structured for students to build incrementally. The
   foundation is already in place:

  - Polished UI: Landing page with hero section, feature cards, login/register
  forms
  - Warm, minimalist design: Paper/ink aesthetic with green accents
  - Routing skeleton: All routes defined (/expenses/add, /edit, /delete, etc.)

  What's missing (intentionally, for students to implement):
  - Database layer (database/db.py is a placeholder)
  - Authentication logic (login/register sessions)
  - CRUD operations for expenses
  - User profile functionality

  Tech Stack

  ┌───────────┬─────────────────────────────────┐
  │   Layer   │           Technology            │
  ├───────────┼─────────────────────────────────┤
  │ Backend   │ Python 3 + Flask                │
  ├───────────┼─────────────────────────────────┤
  │ Database  │ SQLite (via expense_tracker.db) │
  ├───────────┼─────────────────────────────────┤
  │ Templates │ Jinja2                          │
  ├───────────┼─────────────────────────────────┤
  │ Styling   │ Custom CSS with design tokens   │
  └───────────┴─────────────────────────────────┘

  The .gitignore you opened confirms the project tracks:
  - .db files (SQLite database)
  - venv/ (virtual environment)
  - __pycache__/, .pyc files (Python bytecode)
  - .env files (environment variables/secrets)

  In short: it's a teaching tool—students get a working UI and fill in the
  backend logic step by step.
